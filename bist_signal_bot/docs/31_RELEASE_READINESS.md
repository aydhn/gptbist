# Release Readiness

The `release` module ensures the application is in a safe, expected state for a local deployment.

## Core Concepts

* **Release Readiness Score (0-100)**: Evaluated based on importability, security flags, config defaults, and missing optional dependencies.
* **Release Blocker**: Certain issues (like a configured Telegram API token leaking in outputs or `ENABLE_REAL_ORDERS` set to True) will cause the readiness status to become `BLOCKED`.
* **Safe Launch Rehearsal**: Simulates the end-to-end flow without writing destructive data.
* **Manifests**: When you run `release candidate`, it generates a manifest and a Markdown release notes file.

## Expected Checks

* `Imports`: Ensures all standard modules load.
* `Config`: Ensures `.env.example` exists and default settings are safe.
* `Storage`: Asserts the app has write access to the `data/` folder.
* `Safety`: Uses the Forbidden Action Guard to smoke test whether a mock "place order" action is rejected.
* `Runtime`: Validates the dry-run orchestrator.

## Disclaimer

All outputs generated by this system are for research purposes. **Not investment advice. No real orders are sent.**
