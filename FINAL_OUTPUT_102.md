1. **Eklenen/güncellenen dosyalar**: `bist_signal_bot/data_import/models.py`, `adapters.py`, `schema_mapping.py`, `preview.py`, `normalization.py`, `validation.py`, `importer.py`, `chunking.py`, `provenance.py`, `storage.py`, `reporting.py`, `app/data_import_app.py`, test dosyaları ve entegrasyon yer tutucuları (`security/path_guard.py` dahil).
2. **Local Data Import Adapters v1 mimarisi özeti**: Internetsiz, dry-run by default çalışan, chunked-okuma destekleyen ve schema mapping yapabilen bir veri içeri alma sistemidir. Sadece local CSV, JSON, JSONL, Parquet ve SQLite okur; web scraping veya broker kullanımı kesinlikle yoktur.
3. **Modeller**: `ImportSource`, `SchemaMapping`, `ImportJob`, `ImportPreview`, `ImportValidationResult`, `NormalizedImportResult` vb. Pydantic ile tanımlandı. Tüm modeller default "investment advice değildir" disclaimer'ı barındırıyor.
4. **Adapter registry davranışı**: Desteklenen dosya uzantılarını (CSV, JSON, vb.) tespit eder (infer format), Parquet/Excel gibi kütüphanelere optional dependency ile yaklaşır, pandas okumaları ve preview yapar.
5. **Schema mapping engine davranışı**: Sütun isimlerini (case-insensitive ve temel Türkçe terimlerle) hedefe dönüştürür. `symbol`, `date`, `close` gibi zorunlu hedeflerin eksikliğinde `FAIL` veya `WATCH` kararı verir, mapping confidence puanı hesaplar.
6. **Preview builder davranışı**: `max_rows` okuyarak kolon türlerini (`infer_types`) belirler, row sayısını hesaplar. Preview verisini güvenli hale (sanitize) getirerek serialize edilebilir yapar.
7. **Normalizer davranışı**: Sembolleri uppercase yapar, tarihleri deterministik `YYYY-MM-DD` stringine dönüştürür, decimal comma/nokta ayarını ve sayısal dönüşümleri uygular. Gerekli kolonları eksik olan veya kopya (duplicate) kayıtları olan satırları filtreler.
8. **Validation engine davranışı**: Format eksikliğini, dosyanın yokluğunu, preview verisindeki tutarsızlıkları ve en önemlisi "unsafe path" (path traversal) durumunu engeller, `BLOCKED`, `FAIL`, `WATCH` statüleri döndürür.
9. **Local data importer davranışı**: `run_import` dry-run default'tur. Eğer `--confirm` verilirse normalized dosyayı (ör. parquet) çıktı klasörüne yazar. Kalıcı işlemler hep "confirm" flagine bağlıdır.
10. **Chunk reader ve provenance davranışı**: `ImportChunkReader` dosyaları iteratif (lazy) şekilde okuyarak memory korur. `ImportProvenanceBuilder` dosyalar için SHA256 checksum üretir ve lineage adımlarını (from -> to NORMALIZATION) kayıt altına alır.
11. **Data Catalog / Feature Store entegrasyonu**: `register_catalog` çağrısı sembolik eklendi, Feature store placeholder'ı oluşturuldu. İlerleyen fazlar için yapılar hazır.
12. **Orchestrator / Performance entegrasyonu**: Orchestrator, Performance placeholderları hazır. Benchmarking ve batch processing uyumlu bir dizayn yapıldı.
13. **QA / Ops / Docs Hub / Final Handoff entegrasyonu**: İlgili dosyalar oluşturuldu, ayarlar (`QA_INCLUDE_DATA_IMPORT`, `OPS_INCLUDE_DATA_IMPORT`) settings.py içine yazıldı.
14. **Reports / Healthcheck / Doctor / Governance entegrasyonu**: `data-import report` komutu oluşturuldu ve markdown generator yapıldı. Ayarlar ve yer tutucular tanımlandı.
15. **CLI data-import komutları**: `data-import formats`, `preview`, `map`, `validate`, `normalize`, `run`, `jobs`, `report`, `config` komutları implement edildi.
16. **Audit/Notification entegrasyonu**: `core/audit.py` içine yeni `DATA_IMPORT_*` eventleri eklendi, `formatter.py` placeholder'ı oluşturuldu.
17. **README / docs / examples güncellemeleri**: `README.md`'ye "Data Import" başlığı eklendi. `82_LOCAL_DATA_IMPORT_ADAPTERS.md` dokümanı yazıldı ve `data_import_workflow.md` example'ı oluşturuldu.
18. **Test listesi**: Modeller, adapterlar, schema mapping, preview, normalizer, validation, importer, chunking, provenance, storage, reporting ve CLI bileşenleri için 29 adet offline ve mock tabanlı deterministic pytest vakası yazıldı (hepsi PASSED).
19. **Çalıştırma komutları**: Testleri çalıştırmak için `python -m pytest ... -v` ve CLI için `python -m bist_signal_bot data-import ...` scriptleri kullanılabilir.
